# 🌸 Бот для записи клиенток

Telegram-бот с красивым WebApp для записи. Клиентки сами выбирают
дату/время, переносят и отменяют записи через бота. Тебе приходят уведомления
о каждой записи/переносе/отмене. Напоминания клиенткам отправляются
автоматически за день до визита.

## Что внутри

```
booking_bot/
├── main.py              — бот + веб-сервер (всё в одном процессе)
├── database.py          — работа с SQLite (база создаётся сама)
├── webapp_auth.py        — проверка подлинности данных от Telegram
├── config.py             — настройки из переменных окружения
├── webapp/
│   └── index.html        — розовый WebApp (HTML+CSS+JS, без сборки)
├── requirements.txt
├── render.yaml            — для деплоя на Render одним кликом
└── .env.example
```

## Шаг 1. Создать бота в Telegram

1. Напиши **@BotFather** в Telegram → `/newbot`
2. Придумай имя и username (должен кончаться на `bot`)
3. Получишь токен вида `123456:ABC-DEF...` — сохрани, это `BOT_TOKEN`

## Шаг 2. Узнать свой Telegram ID

1. Напиши **@userinfobot** в Telegram
2. Он покажет твой `id` — это `ADMIN_IDS` (можно указать несколько через запятую,
   если админов несколько)

## Шаг 3. Деплой на Render (бесплатно)

1. Зарегистрируйся на [render.com](https://render.com) (можно через GitHub)
2. Залей этот проект в свой репозиторий на GitHub
   (создай новый репозиторий → загрузи туда все файлы из `booking_bot/`)
3. На Render: **New → Blueprint** → выбери свой репозиторий
   — Render сам увидит `render.yaml` и предложит создать сервис
4. На странице сервиса заполни переменные окружения (Environment):
   - `BOT_TOKEN` — токен от BotFather
   - `ADMIN_IDS` — твой id
   - `WEBAPP_URL` — оставь пока пустым, заполнишь после деплоя
   - `BASE_URL` — тоже пока пустым
5. Нажми **Deploy** — через пару минут получишь адрес вида
   `https://booking-bot-xxxx.onrender.com`
6. Вернись в Environment и заполни:
   - `WEBAPP_URL = https://booking-bot-xxxx.onrender.com/webapp/`
   - `BASE_URL = https://booking-bot-xxxx.onrender.com`
7. Сервис автоматически перезапустится с новыми настройками — готово!

> ⚠️ **Важно про Telegram WebApp**: Telegram требует, чтобы адрес WebApp был
> на **HTTPS**. Render бесплатно выдаёт HTTPS из коробки, так что всё ок —
> просто не убирай `https://` из ссылки.

## Шаг 4. Проверка

1. Напиши своему боту `/start`
2. Должна появиться кнопка «🌸 Записаться» — нажми
3. Откроется розовый WebApp с календарём

**Если слотов нет** — это нормально, их пока никто не добавил (см. Шаг 5).

## Шаг 5. Добавление слотов (твоя админка прямо в боте)

Команды для тебя как администратора (пиши их боту в личных сообщениях):

```
/admin                — список всех команд
/addslot 05.07.2026 14:30          — добавить один слот
/addday 05.07.2026 10:00 18:00 60  — добавить слоты на весь день
                                      (с 10:00 до 18:00 каждый час)
/today                 — кто записан на сегодня
/week                  — кто записан на неделю вперёд
```

Пример: если работаешь по будням с 10 до 19 с шагом 90 минут —
просто пиши `/addday` на каждый рабочий день. Можно автоматизировать
позже (скажи мне, если понадобится — добавлю авто-генерацию на месяц вперёд).

## ⚠️ Важные технические особенности бесплатного хостинга

- **Render free tier "засыпает"** после ~15 минут без запросов и
  "просыпается" 30-60 секунд на первое сообщение. Бот сам пингует себя
  каждые 10 минут (`self_ping_loop` в `main.py`), это снижает частоту
  засыпания, но не убирает её полностью — это ограничение бесплатного тарифа,
  а не баг.
- **SQLite-файл** (`booking.db`) живёт на диске сервиса. На Render free tier
  диск **не persistent** между деплоями — если будешь обновлять код через
  git push, файл базы может пересоздаться с нуля. Для учебного/небольшого
  использования это ок, но если нужна гарантированная сохранность записей —
  скажи, подключим бесплатный Render PostgreSQL (тоже есть free tier) или
  Render Persistent Disk (платный, $1/мес за 1GB).
- Если хочешь вообще без простоев и рисков — это уже платный план Render
  (~$7/мес) или альтернативные платформы (Railway, Fly.io — тоже со своими
  бесплатными лимитами и нюансами).

## Локальный запуск (для тестов на своём компьютере)

```bash
cd booking_bot
python -m venv venv
source venv/bin/activate   # на Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env        # и заполни своими значениями
export $(cat .env | xargs)  # на Windows используй другой способ загрузки .env
python main.py
```

Для локального теста WebApp Telegram всё равно потребует HTTPS-адрес —
проще сразу тестировать на Render, либо использовать `ngrok` для туннеля.

## Как это расширять дальше

Идеи, если захочешь развивать бота:
- авто-генерация слотов на месяц вперёд по шаблону рабочей недели
- разные длительности услуг (например, стрижка 60 мин, окрашивание 120 мин)
- учёт оплаты/предоплаты
- статистика: сколько клиенток в месяц, повторные визиты
- экспорт записей в Excel

Просто напиши, что нужно — добавим.
